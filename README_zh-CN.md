# ⚓ 港口大师 2 · React + TypeScript 版

> 🌐 **双语文档** | [🇬🇧 View English](README.md)

这是港口大师 2 的一次彻底重制。原先的 Python 服务端加单文件 HTML 前端，被完整改写成了一个类型完备的 Node 后端和一个 React 前端。游戏内容、规则玩法、双语界面都和原作一模一样，只是把当初的两个巨型文件，整理成了一个清爽的 monorepo。

---

## 📖 1. 游戏简介

欢迎回到海上丝绸之路。两位船长共同经营同一段航程，始终停留在同一回合、同一阶段。你们既要比拼声望，也会在途中互通有无，交换货物、金币和情报。从航海家的罗盘里抽取一项福缘，向牙行购买密语，给旗舰加装模块，并在每段航程里都为工资、维护费和税款留足现金（轻松 8 回合、标准 12 回合、高难 16 回合）。声望更高的一方获胜。

这一版保留了原作的全部内容，只更换了底层引擎：

- 🧩 **一个游戏，两个应用，一份共享核心。** Node 的 WebSocket 服务端掌握全部权威游戏逻辑，React 前端负责渲染。两端都引用同一份 `packages/shared` 里的数据表和协议类型，因此永远不会各执一词。
- 🔤 **全程 TypeScript**，从难度阶梯到互市防刷校验，都开启了严格的编译选项。
- 🧪 **一套真正的测试。** 214 个单元与集成测试覆盖了各项公式、阶段状态机、密码哈希、大厅，以及一次完整的 Socket 往返流程。
- 🌍 **英文（默认）与简体中文**，每个界面都能用 🌐 按钮随时切换，登录页和大厅页也不例外。
- 🪶 **文件小而清晰。** 原作 1862 行的服务端和 4098 行的 HTML，被拆成了约一百个职责单一的小文件，每个都对应原作的某一处。

如果你从没玩过，这一版的玩法和港口大师 2 完全一致。下文既讲怎么运行，也讲怎么游玩。

## 🛠️ 2. 安装与运行

### ✅ 前置要求

- 主机上需要 **Node.js 20.19 及以上，或 22.12 及以上**（Vite 构建需要较新的运行时）。
- **npm**（随 Node 一同安装）。
- Windows、macOS 或 Linux 均可。玩家本人只需要一个现代浏览器。

### 🚀 快速上手（单端口，适合分享）

正式游玩，以及通过互联网分享，都推荐这种方式。

```bash
npm install        # 一次性安装所有工作区
npm run serve      # 先构建前端，再用单一端口托管整个游戏
```

在浏览器里打开 **http://localhost:8080**，注册账号并登录。网页和 WebSocket 共用同一个端口，所以一条隧道就能带起整个游戏：

```bash
ngrok http 8080
```

把 ngrok 给出的地址发给朋友即可。https 与 wss 的适配是自动完成的，无需任何额外配置。

### 🧑‍💻 开发模式（热更新，双端口）

改代码时通常希望有热更新，于是前端和后端分开运行。开两个终端：

```bash
npm run dev:server   # Node 后端跑在 :8080，改动后自动重启
npm run dev:client   # Vite 开发服务器跑在 :5173，模块热替换
```

打开 **http://localhost:5173**。Vite 会把 `/ws` 这条 WebSocket 转发给后端，所以浏览器始终只对着一个地址。

### 🎯 两个浏览器，一局游戏

港口大师 2 是双人对局，需要两个已登录的会话才能离开登录页。请用两个不同的浏览器，或者一个普通窗口加一个无痕窗口，各自注册一个账号。

## 🗂️ 3. 项目结构

整个项目是一个 npm workspaces 的 monorepo，三个包各司其职：

```
ReactPM2/
├── packages/shared/      # 两端共享的唯一事实来源
│   └── src/
│       ├── data/         # 游戏数据表：配方、价格、工资、福缘、
│       │                 # 模块、季风、港口、难度阶梯……
│       ├── domain/       # 核心类型：工人、互市订单、密语线索、卡牌……
│       └── protocol/     # 通信协议：PlayerGameState、各类消息
│
├── apps/server/          # 权威的 Node 游戏服务端（无界面）
│   ├── src/
│   │   ├── game/         # PlayerGame 及其委托的纯函数规则
│   │   │                 # （成本、生产、海盗、模块、发牌）
│   │   ├── session/      # SharedSession，双人阶段状态机
│   │   ├── actions/      # 每个游戏内动作一个处理函数（采购、互市、雇佣……）
│   │   ├── lobby/        # 在线名册、邀请、聊天
│   │   ├── auth/         # PBKDF2 密码哈希、用户存储
│   │   ├── ws/           # WebSocket 连接循环与消息路由
│   │   ├── http/         # 为构建后的前端做静态文件服务
│   │   └── main.ts       # 把一切串起来的入口
│   ├── tests/            # 214 个 vitest 测试，目录结构与 src/ 对应
│   └── data/users.json   # 账号存储（运行时创建和更新）
│
└── apps/client/          # React 前端（Vite）
    └── src/
        ├── i18n/         # 双语层：所有文案、手册正文、
        │                 # 服务端日志翻译表、物品名称……
        ├── ws/           # 唯一的 WebSocket 钩子
        ├── state/        # 会话、提示、观战的 React 上下文
        ├── components/   # 布局、登录、大厅、各面板、各阶段界面，
        │                 # 以及弹窗、手册、聊天、提示气泡、观战等系统
        └── styles/       # 全局样式表
```

核心原则是：凡是两端必须保持一致的东西，都放在 `packages/shared` 里被引用，而绝不复制。这是防止前后端在某个价格或某条规则上产生分歧的主要手段。

## 🎮 4. 游戏机制

轻松 **8 段航程**（回合），标准 **12 段**，高难 **16 段**，每段航程都要经过 **8 个阶段**。任何阶段都要等双方都确认后才会推进。

| 阶段 | 内容 |
|:---|:---|
| **⚓ 启航** | 确认开始本回合。从第 2 回合起，这一页还会回顾上一回合的结算。 |
| **🧭 福缘** | 罗盘随机给你发出若干福缘，对方拿到的是另一手。锁定其中一项，只在本回合生效。 |
| **🛒 采购** | 从补给卡里购买原料和成品。牙行密语面板就在最上方，出售关于未来需求的情报。 |
| **🤝 互市** | 与伙伴交易。发布一条「我出某物换某物」的提案，对方一接受即刻成交。 |
| **👥 工匠** | 雇佣或解雇工匠，并分配生产任务。材料会立即扣除。 |
| **📦 贸易** | 用库存交付港口订单。你买到的线索会在这里兑现成带 🗣️ 标记的稳定订单。 |
| **🔧 结算** | 产出自动入库、工资自动支付，然后你要付 15 金币的船队维护费。付不起就破产。 |
| **🚢 船坞** | 想升级就升级商船（0 到 3 级），抽取模块，然后结束本段航程。 |

### ⚖️ 难度模式

每局都从 **轻松** 开始。发出邀请的一方挑选难度，另一方看过说明并同意后才开局，因此两位船长始终在同一难度下航行。

- **轻松（8 回合）** 让整段航程都停留在基础货物之内：三种原料、四种入门成品、前三类工匠，以及与之配套的福缘、模块、港口和天象。市场永远不会过度拥挤，是从容上手的好起点。
- **标准（12 回合）** 以更紧凑的节奏开放完整贸易，丝路特许在第 4、8 回合分批解锁，但不会出现暗通海盗的牙行。
- **高难（16 回合）** 是开启了可疑牙行机制的完整挑战，在第 6、10 回合分批解锁。需要打理的事务多得多，对舱位和金币的争夺也激烈许多。

### 📦 原料与成品

你先购入原料，由工匠加工成成品，再用成品交付订单。成品的价值远高于原料，所以「低买、加工、高卖」这个循环才是利润所在。商品在其特产港口更便宜，价格也会随天象浮动。

### 🧭 福缘

每回合航海家的罗盘都会给你单独发一手 4 张随机增益，你锁定其中一项，只在本回合有效。效果从直接给金币，到运费减半、雇佣更便宜、额外多一张订单，或免费获得一条牙行密语线索，不一而足。

### 🔧 船舶模块

升级商船即可解锁模块槽位（每级一个，最多 3 个）。每个船坞阶段，你都可以抽出固定的一批三个模块并安装其中一个，从走私暗舱到超载引擎都有。每回合还能把这一批换一次。

### 💰 税收与财务

成品订单大约缴 5% 的增值税，每回合结束时再按本回合净利润缴约 10% 的所得税（持「免税令」福缘时为 5%）。无论赚没赚到钱，工资和 15 金币的维护费每回合都得付，所以现金流永远是第一位的。

### 🌐 联机机制

账号采用加盐的 PBKDF2 哈希存储。大厅会列出谁在线，邀请会带上所选难度，每分钟一次的冷却让节奏不至于太乱。即便掉线，你的账号和正在进行的会话也会留在服务端，因此只要伙伴还在，你重新登录就能接着玩。

## 💡 6. 策略建议

- **现金流压倒一切。** 工资加维护费每回合都要付，付不起的后果远比少赚一点严重得多。左侧面板会实时核算你本回合应付的款项，时刻盯着它。
- **卖成品，别卖原料。** 一个香囊的价值远高于做它所用的材料。
- **购买密语。** 每条线索都会在本回合后段兑现成稳定订单，所以提前备货就能稳赚；装上牙行网络模块还能让密语更便宜。
- **留意天象。** 每回合的季风会抬高某些港口的报酬、压低某些采购价，也会改变海盗风险。
- **多和伙伴沟通。** 你们能看到彼此的货物和金币，所以善用互市阶段和聊天，互相补缺。

## 🏆 7. 游戏终局与评级

当最后一段航程结算完毕，你的声望（每笔已交付订单的净利润累计）会换来一个商人评级。由于航程越长赚得越多，每个难度都有各自的门槛。最高的 **丝绸之路霸主** 评级要求声望达到 **轻松 1200 以上**、**标准 3000 以上**、**高难 6000 以上**。破产的船长依然可以打开观战窗口，看着伙伴走完航程；等两位船长都结束后，任意一方都能重新起航。

## 🛡️ 8. 故障排除

- **运行脚本时报 `Cannot find module ... .bin/...`。** `node_modules/.bin` 里的符号链接可能被某些云同步工具压成了普通文件（同步中的桌面目录上的 iCloud Drive 是常见元凶）。用 `rm -rf node_modules/.bin && npm install` 重建即可。
- **页面能打开但没有样式。** 确认你是通过 `npm run dev:client` 启动，或用 `npm run build` 构建过前端。样式表由 Vite 打包，直接打开裸文件是加载不到的。
- **过不了登录页。** 这需要两个账号。打开第二个浏览器或无痕窗口，再注册一位船长。
- **服务端重启过。** 账号会留在 `apps/server/data/users.json` 里，但正在进行的航程存在内存中，重启即丢失，请重开一局。
- **想通过互联网分享。** 用单端口构建（`npm run serve`），再把那一个端口隧道出去（`ngrok http 8080`），让网页和 WebSocket 共用同一个 https 来源。
